# Superconductor Mobile — v1.1 Mac Menu Bar Companion (Implementation Plan)

**Date:** 2026-06-18  
**Status:** Ready for execution  
**Owner of this plan:** Third-party companion (we do not control Superconductor.app)  
**Related:** [v1.1-roadmap.md](./2026-06-18-v1.1-roadmap.md), [design.md](./2026-06-18-superconductor-mobile-design.md)

## Goal

Deliver the recommended v1.1 Mac Companion path: replace `bun run bridge:start` with a native macOS menu bar / status item application.

Users install and run (or set to launch at login) one small companion app on their Mac. It provides:
- The mobile REST + WebSocket bridge surface
- Beautiful QR code for instant iPhone pairing
- Status, token management, host info

The iPhone experience remains unchanged (or gets a small optional QR scanner improvement).

## Why this direction (third-party reality)

- We cannot modify `/Applications/Superconductor.app`.
- Superconductor.app (native) + `sc` CLI + local Unix socket is the only supported interface.
- The current Bun bridge is a thin, correct adapter. We keep the public HTTP/WS contract 100% stable.
- A first-class Swift menu bar app makes the bridge "just part of the Mac" for users.

## Principles (non-negotiable)

1. Official `sc` / local API only — no direct socket access or internals.
2. Contract stability: iOS code and protocol (PairingPayload, /v1/* routes) must continue to work without changes (or with additive-only changes).
3. KISS + YAGNI: Start by **managing the existing Bun adapter**. Provide excellent UI. Defer full Swift server port.
4. Native macOS: Use Swift + AppKit/SwiftUI. Follow standard menu bar conventions (no Dock icon by default, SF Symbols, calm UI).
5. Security: Same token model. Prefer Tailscale. No public exposure.

## Scope

### In Scope (MVP for this plan)

## Architecture

```
┌────────────────────────────┐
│ iPhone (SwiftUI)           │
│ (existing PairingView etc) │
└──────────────┬─────────────┘
               │ HTTPS/WS + Bearer (same contract)
               ▼
┌──────────────────────────────────────────────┐
│ SuperconductorMobileCompanion.app (Swift)    │
│  • NSStatusItem (menu bar)                   │
│  • NSPopover + SwiftUI content (QR + UI)     │
│  • Child Process management                  │
│  • Config read/write (bridge.json)           │
│  • PairingPayload generation (local)         │
└──────────────┬───────────────────────────────┘
               │ spawns + monitors
               ▼
┌──────────────────────────────────────────────┐
│ Bridge server (existing Bun or bundled)      │
│  (apps/bridge/src/server.ts)                 │
│  Talks to Superconductor via `sc` + socket   │
└──────────────────────────────────────────────┘
```

The companion owns the **user-facing Mac experience**. The actual protocol adapter stays the proven Bun code.

## Key Decisions

- **Manage, don't rewrite yet**: Launch the existing bridge as a child `Process`. This gives fast value and reuses all current logic (agents, streams, worktree stub, etc.).
- **Standalone bundling**: Use `bun build --compile` to produce a single executable for the server. Embed it in the app bundle for distribution.
- **Payload generation**: Companion constructs `PairingPayload` directly from its config (avoids auth chicken-egg for QR). Matches the shape in `packages/protocol`.
- **No new public surface**: The companion does not add routes. It only provides UI + process supervision.
- **Dev vs Release**:
  - Dev: Use `bun` from PATH + source.
  - Release: Use the pre-compiled binary inside the bundle.
- **UI style**: Simple, native, Apple-like. Use `glassEffect` where appropriate (macOS 15+), but keep functional.

### 2. Port / reimplement minimal config logic in Swift (`CompanionConfig.swift`)

Port the important parts from `apps/bridge/src/config.ts`:

```swift
import Foundation
import CryptoKit

struct CompanionConfig: Codable {
    var token: String
    var fingerprint: String
    var port: Int
    var bind: String
}

enum CompanionConfigManager {
    static let configDir = FileManager.default.homeDirectoryForCurrentUser
        .appendingPathComponent(".superconductor-mobile")
    static let configPath = configDir.appendingPathComponent("bridge.json")

    static func loadOrCreate() -> CompanionConfig {
        try? FileManager.default.createDirectory(at: configDir, withIntermediateDirectories: true)
        
        if let data = try? Data(contentsOf: configPath),
           let config = try? JSONDecoder().decode(CompanionConfig.self, from: data) {
            return config
        }
        
        let token = Data((0..<32).map { _ in UInt8.random(in: 0...255) }).base64EncodedString()
            .replacingOccurrences(of: "+", with: "-")
            .replacingOccurrences(of: "/", with: "_")
            .replacingOccurrences(of: "=", with: "")
        
        let fingerprint = Insecure.SHA256.hash(data: Data(token.utf8))
            .map { String(format: "%02x", $0) }
            .joined()
            .prefix(16)
            .description
        
        let config = CompanionConfig(
            token: token,
            fingerprint: String(fingerprint),
            port: 9477,
            bind: "0.0.0.0"
        )
        
        if let data = try? JSONEncoder().encode(config) {
            try? data.write(to: configPath, options: .atomic)
        }
        return config
    }
    
    static func regenerate() -> CompanionConfig {
        try? FileManager.default.removeItem(at: configPath)
        return loadOrCreate()
    }
}
```

### 4. Bridge process management (`BridgeProcessManager.swift`)

```swift
@MainActor
final class BridgeProcessManager: ObservableObject {
    @Published var isRunning = false
    private var process: Process?
    
    func start(config: CompanionConfig) {
        guard process == nil else { return }
        
        let proc = Process()
        // Dev path or bundled path
        let bridgeExec = Bundle.main.url(forResource: "bridge-server", withExtension: nil)?.path 
            ?? "/opt/homebrew/bin/bun"   // dev fallback
        
        if bridgeExec.contains("bun") {
            proc.executableURL = URL(fileURLWithPath: bridgeExec)
            proc.arguments = ["run", "--cwd", /* path to apps/bridge in dev */ "src/server.ts"]
        } else {
            proc.executableURL = URL(fileURLWithPath: bridgeExec)
        }
        
        proc.environment = ProcessInfo.processInfo.environment.merging([
            "SC_MOBILE_BRIDGE_PORT": String(config.port),
            "SC_MOBILE_BRIDGE_BIND": config.bind
        ]) { _, new in new }
        
        // Optional: capture stdout/stderr to logs
        try? proc.run()
        self.process = proc
        isRunning = true
        
        // Observe termination
        proc.terminationHandler = { _ in
            Task { @MainActor in self.isRunning = false }
        }
    }
    
    func stop() {
        process?.terminate()
        process = nil
        isRunning = false
    }
}
```

For production build, the receiving agent will add a build phase that runs `bun build ... --compile` and places `bridge-server` next to the executable or in Resources.

### 5. Menu bar + Popover UI

In `AppDelegate.swift` (or a dedicated controller):

```swift
class AppDelegate: NSObject, NSApplicationDelegate {
    var statusItem: NSStatusItem!
    var popover = NSPopover()
    
    func applicationDidFinishLaunching(_ notification: Notification) {
        NSApp.setActivationPolicy(.accessory)
        
        statusItem = NSStatusBar.system.statusItem(withLength: NSStatusItem.squareLength)
        if let button = statusItem.button {
            button.image = NSImage(systemSymbolName: "iphone.gen2", accessibilityDescription: "Superconductor")
            button.target = self
            button.action = #selector(togglePopover(_:))
        }
        
        let contentView = PairingPopoverView()
            .environmentObject(bridgeManager)
        popover.contentViewController = NSHostingController(rootView: contentView)
        popover.behavior = .transient
    }
    
    @objc func togglePopover(_ sender: Any?) {
        if popover.isShown {
            popover.performClose(sender)
        } else if let button = statusItem.button {
            popover.show(relativeTo: button.bounds, of: button, preferredEdge: .minY)
        }
    }
}
```

### 6. SwiftUI Popover Content (`PairingPopoverView.swift`)

```swift
struct PairingPopoverView: View {
    @EnvironmentObject var manager: BridgeProcessManager
    @State private var config = CompanionConfigManager.loadOrCreate()
    @State private var host = "loading..."
    
    var body: some View {
        VStack(alignment: .leading, spacing: 12) {
            Text("Superconductor Mobile").font(.headline)
            
            if let qrImage = generateQRImage() {
                Image(nsImage: qrImage)
                    .resizable()
                    .frame(width: 180, height: 180)
            }
            
            Text("Scan with the Superconductor iOS app")
                .font(.caption)
            
            HStack {
                Button("Regenerate Token") {
                    config = CompanionConfigManager.regenerate()
                    // restart bridge with new config
                }
                Button("Copy Token") { /* pasteboard */ }
            }
            
            Text("Host: \(host)  Port: \(config.port)")
                .font(.caption2).foregroundStyle(.secondary)
            
            Button(manager.isRunning ? "Stop Bridge" : "Start Bridge") {
                if manager.isRunning { manager.stop() } else { manager.start(config: config) }
            }
        }
        .padding()
        .frame(width: 240)
        .onAppear { host = bestHost() }
    }
    
    func generateQRImage() -> NSImage? {
        // Build payload JSON string
        let payload = PairingPayload(version: 1, host: host, port: config.port, token: config.token, fingerprint: config.fingerprint, tls: false)
        let json = try! JSONEncoder().encode(payload)
        let string = String(data: json, encoding: .utf8)!
        
        // CIFilter QR generation (see earlier sketch)
        guard let filter = CIFilter(name: "CIQRCodeGenerator") else { return nil }
        filter.setValue(string.data(using: .utf8), forKey: "inputMessage")
        // scale + NSImage conversion...
    }
}
```

## Risks & Mitigations

- Bun not present on user machines → bundle standalone binary.
- Multiple bridge instances → companion is the single source of truth for the process.
- Port conflicts → allow config override.
- Process crashes → companion can auto-restart or show error in UI.

## Handoff Notes for Executor Agent

- Prioritize getting a working menu bar + QR that produces a usable token first.
- The Bun child process launch is the quickest path to full functionality.
- Once the UI and process management works, the receiving agent can explore embedding a pure-Swift server if desired.
- Keep all changes additive to the existing bridge contract.

## Success Criteria

A user can:
1. Build / download the companion app.
2. Launch it once.
3. See menu bar icon + QR.
4. Scan or copy values → iPhone pairs successfully and controls agents.
